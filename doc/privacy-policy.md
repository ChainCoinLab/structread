# Privacy Policy for StructRead

Last updated: 2026-04-21

## Overview

StructRead is a Chrome extension that helps users understand complex English sentences by analyzing selected text, highlighting grammatical structure, and saving vocabulary items for later review.

This Privacy Policy explains what data the product processes, how it is used, and when it may be shared.

## Data We Process

StructRead may process the following categories of data:

- text that a user explicitly selects for sentence analysis
- words that a user explicitly selects for word lookup
- vocabulary items saved by the user
- extension preferences such as reading mode, highlight colors, API endpoint settings, and provider settings
- API provider settings entered by the user, such as model selection and API key values

## How We Use Data

We use this data only to provide the extension's user-facing functionality, including:

- analyzing selected English sentences
- returning sentence structure explanations
- highlighting sentence components on the current page
- looking up word definitions and pronunciation data
- saving vocabulary for later review or export
- remembering extension settings and preferred providers

## Data Sharing

Selected text or selected words may be transmitted to:

- the developer's backend service that performs or coordinates sentence and word analysis
- third-party AI model providers that the user explicitly configures for analysis

We do not sell user data.

We do not use user data for advertising or profiling.

We do not transfer user data to data brokers or information resellers.

## Local Storage

The extension stores some information locally in Chrome storage, including:

- saved vocabulary
- highlight color preferences
- reading mode selection
- provider configuration
- active provider selection
- endpoint configuration

## Human Access

We do not allow humans to read user-selected text except when necessary for security, legal compliance, or user-requested support involving specific data.

## Data Retention

Locally stored data remains in the user's browser until the user deletes it, clears extension storage, or uninstalls the extension.

Server-side retention, if any, should be limited to what is necessary to provide the service securely and reliably. If you operate the production backend, document your exact log and retention policy before launch.

## Security

Production deployments should use HTTPS for all extension-to-server communication. API keys and any transmitted text should be handled securely and only for the extension's stated purpose.

## User Choices

Users can:

- choose whether to analyze selected text
- choose whether to save vocabulary
- remove saved vocabulary from local storage
- export saved vocabulary
- configure or remove AI providers
- uninstall the extension at any time
